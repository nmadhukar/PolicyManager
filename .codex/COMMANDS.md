# Codex Command Map

These commands are procedural docs, not shell scripts yet.

Use:

- `.ai/commands/plan-phase.md` to expand a phase into tickets.
- `.ai/commands/build-ticket.md` to implement one approved ticket.
- `.ai/commands/verify-phase.md` to close a phase.
- `.ai/commands/migrate.md` for Prisma/PostgreSQL changes.
- `.ai/commands/db-schema-check.md` to prove objects are in `policytracker`.
- `.ai/commands/rbac-check.md` for permission changes.
- `.ai/commands/storage-check.md` for S3/MinIO changes.
- `.ai/commands/release-check.md` before production release.

Do not infer missing commands. If the app has not been scaffolded yet, report that runtime commands are not available.
