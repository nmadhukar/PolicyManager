# Ticket Status Conventions

Use these statuses for backlog and ticket tracking.

```text
draft
ready
in_progress
blocked
review
accepted
deferred
cancelled
```

## Status Rules

- `draft`: incomplete task details.
- `ready`: has acceptance criteria, tests, and commands.
- `in_progress`: actively being worked.
- `blocked`: cannot continue without user decision or external dependency.
- `review`: implementation complete, awaiting review.
- `accepted`: done evidence accepted.
- `deferred`: intentionally postponed.
- `cancelled`: no longer needed.

## Required Done Evidence

- Files changed.
- Tests/commands run.
- Documentation updates.
- Remaining risks.
- Follow-ups.
