# Skill: Task Planning

## Purpose

Turn a phase or feature idea into a small ticket that can be implemented and verified.

## Use When

- Creating tickets.
- Refining backlog.
- Before implementation.

## Procedure

1. Identify active phase.
2. Define one concrete goal.
3. State non-goals.
4. Define user workflow.
5. Define acceptance criteria.
6. Identify data/API/UI/security/audit impact.
7. Define tests.
8. Define commands.
9. Define rollback.
10. Confirm Definition of Done.

## Output

Use `.ai/tasks/TASK_TEMPLATE.md`.

## Sizing Rule

A ticket should be roughly half-day to one-day sized for an AI coding agent and independently verifiable.

If a ticket touches more than two major boundaries, split it.
