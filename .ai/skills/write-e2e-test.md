# Skill: Write E2E Test

## Purpose

Add end-to-end coverage for critical user workflows.

## Use When

- Closing a workflow-heavy ticket.
- Closing a phase.

## Procedure

1. Define the user role.
2. Define the workflow.
3. Seed required data.
4. Exercise UI or API path.
5. Assert visible result and persisted effect.
6. Assert negative path where relevant.
7. Keep test deterministic.
8. Document any required local services.

## Priority Workflows

- Login.
- Upload document.
- Upload new version.
- Download with audit.
- Review and attest.
- Export cover page.
- API client read.
