# Skill: UI Workflow

## Purpose

Build frontend screens that support real clinic workflows.

## Use When

- Adding pages.
- Adding forms.
- Adding dashboards.
- Adding admin screens.

## Procedure

1. Identify user role.
2. Identify primary task.
3. Define route.
4. Define data loading.
5. Define success state.
6. Define empty state.
7. Define validation errors.
8. Define 401 and 403 handling.
9. Check mobile and desktop layout.
10. Add e2e coverage for critical workflows.

## Design Rules

- Build the usable app, not a marketing page.
- Use clear status labels for lifecycle states.
- Use forms and tables where they help repeated admin work.
- Do not rely on hidden buttons for security.

## Output

- Route.
- Workflow summary.
- States covered.
- E2E/manual verification.
