# Command: /build-ticket

## Purpose

Implement one approved ticket with TDD and verification.

## Input

```text
/build-ticket <ticket-id>
```

## Procedure

1. Read `AGENTS.md`.
2. Read the ticket.
3. Run repo cartography.
4. Confirm scope and non-goals.
5. Choose required agents and skills.
6. Write failing test for first acceptance criterion.
7. Implement minimal code.
8. Add negative tests.
9. Run focused tests.
10. Run relevant regression commands.
11. Update docs.
12. Record done evidence.

## Required Final Evidence

- Files changed.
- Tests added.
- Commands run.
- Risks.
- Follow-ups.

## Stop Conditions

Stop if the ticket is not ready, too broad, or would violate a phase gate.
